# Overlays

Overlays let you inject static JSON data into specific route responses — either merged on top of generated fake data, or as a complete replacement.

This is useful for:

- **Pinning specific fields** — always return a known user ID or a specific product for a given endpoint.
- **Special cases** — serve a fixed "featured" item at `/products/featured` while all other `/products/:id` routes return fakes.
- **Seeding UI state** — return a pre-built object that exercises a specific UI state (e.g., a user with a pending payment, an order in error state).

---

## Configuration

Add the `overlays` array to your `chameleon.config.ts`:

```typescript
import { defineConfig } from "@chameleon/core";

export default defineConfig({
  schemas: {
    openapi: "./chameleon/schemas/api.openapi.yaml",
  },
  overlays: [
    {
      path:     "/users/me",
      method:   "GET",
      file:     "./chameleon/data/me.json",
      strategy: "deep-merge",
    },
    {
      path:     "/pets/featured",
      method:   "GET",
      file:     "./chameleon/data/featured-pet.json",
      strategy: "replace",
    },
  ],
});
```

### Entry options

| Option | Type | Default | Description |
|---|---|---|---|
| `path` | `string` | — | Exact request path to match |
| `method` | `string` | — | HTTP method (case-insensitive) |
| `file` | `string` | — | Path to a JSON file |
| `strategy` | `"deep-merge" \| "replace"` | `"deep-merge"` | How to apply the overlay |

---

## Strategies

### `deep-merge`

The JSON file is deep-merged on top of the generated fake response. Only the fields present in the file are overridden; all other fields are still generated by Faker.js.

Generated fake:

```json
{
  "id": "4f8a1b2c",
  "name": "Fluffy",
  "status": "available",
  "tags": ["small", "friendly"]
}
```

Overlay file (`featured-pet.json`):

```json
{
  "id": "featured-001",
  "name": "Rex"
}
```

Result:

```json
{
  "id": "featured-001",
  "name": "Rex",
  "status": "available",
  "tags": ["small", "friendly"]
}
```

### `replace`

The JSON file completely replaces the generated response. No fake data is generated for this route.

```json
{
  "id": "featured-001",
  "name": "Rex",
  "status": "adopted",
  "tags": ["large", "trained"]
}
```

## Example: pinning the "current user"

A common pattern is to have `/users/me` return a known user while all other `/users/:id` routes return fakes.

```typescript
overlays: [
  {
    path:     "/users/me",
    method:   "GET",
    file:     "./chameleon/data/me.json",
    strategy: "deep-merge",
  },
],
```

`chameleon/data/me.json`:

```json
{
  "id": "usr_current",
  "email": "demo@example.com",
  "role": "admin",
  "plan": "pro"
}
```

The rest of the user object (name, avatar, timestamps, etc.) is still generated by Faker.js.

---

## Relationship with static JSON schemas

If you load a directory as a `static` schema source, each JSON file in that directory is served as a fixed response. Overlays are different — they are applied on top of routes defined by other schema types (OpenAPI, GraphQL, Postman, Pact).

Use overlays when you want to override a specific route that comes from a parsed schema. Use `schemas.static` when the JSON file _is_ the schema.
