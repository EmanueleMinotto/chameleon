# Chameleon

Chameleon is an open-source mock/faker server that generates realistic API responses directly from your existing schemas — no code changes required. Point it at an OpenAPI spec, a GraphQL SDL, a Postman collection, or a Pact contract, and it starts serving convincing fake responses immediately.

## Why Chameleon?

- **Zero schema modification** — annotations live in separate YAML files, your schemas stay untouched.
- **Realistic data** — responses are generated by [Faker.js](https://fakerjs.dev/), with automatic inference from field names, types, and formats.
- **Production parity** — the same server runs locally via the CLI and on Vercel via a serverless function, using identical logic.
- **Flexible control** — switch between mock and a real upstream API per-request with a single HTTP header.

## Feature overview

| Feature                  | Description                                                                            |
| ------------------------ | -------------------------------------------------------------------------------------- |
| **Multi-schema support** | OpenAPI v2/v3, GraphQL SDL, Postman Collections, Pact contracts, static JSON           |
| **Annotation system**    | YAML files map schema fields to Faker.js methods or custom TypeScript generators       |
| **Auto-inference**       | Missing annotations are inferred from field type, format, and name                     |
| **Static overlays**      | Deep-merge static JSON files on top of generated responses                             |
| **Proxy mode**           | One HTTP header switches between mock and a real upstream server                       |
| **Federation**           | Delegate specific routes to other Chameleon instances                                  |
| **Hoppscotch explorer**  | Built-in endpoint browser, importable into Hoppscotch with one click                   |
| **Vercel deploy**        | One-click deploy — `chameleon build` pre-compiles everything for cold-start efficiency |
| **Hot reload**           | `--watch` mode reloads schemas and annotations without restarting the server           |
