# Project Structure

```
.
├── chameleon/                  ← your working area (edit this)
│   ├── schemas/                ← schema files (OpenAPI, GraphQL, Postman, Pact)
│   │   ├── api.openapi.yaml
│   │   ├── schema.graphql
│   │   ├── collection.postman.json
│   │   └── contracts/
│   ├── annotations/            ← annotation YAML files (supports glob)
│   │   ├── users.annotations.yml
│   │   └── orders.annotations.yml
│   ├── generators/             ← custom TypeScript generator functions
│   │   └── myField.ts
│   ├── data/                   ← static JSON files for overlays
│   │   └── featured-item.json
│   └── chameleon.config.ts     ← main configuration file
│
├── packages/                   ← Chameleon source (do not edit)
│   ├── core/                   ← schema parsers, annotation resolver, generators
│   ├── server/                 ← Hono server, middleware, routes
│   ├── cli/                    ← CLI commands (serve, build, validate, annotate, init)
│   └── vercel/                 ← Vercel adapter
│
├── api/
│   └── chameleon.ts            ← Vercel serverless entry point (do not edit)
│
└── vercel.json                 ← Vercel routing config (do not edit)
```

## The `chameleon/` directory

This is the only directory you need to work in. Everything in `packages/`, `api/`, and `vercel.json` is part of the Chameleon runtime and should not be modified.

### `schemas/`

Place your schema files here. Chameleon supports multiple schemas simultaneously — a request is matched against all loaded routes and the first match wins. See [Schemas](Schemas) for supported formats and configuration.

### `annotations/`

YAML files that control how fake data is generated for each field. You can split annotations across multiple files — they are deep-merged in alphabetical order. Glob patterns are supported in the config:

```typescript
annotations: "./chameleon/annotations/**/*.annotations.yml";
```

See [Annotations](Annotations) for the full syntax.

### `generators/`

TypeScript files that export a default function used as a custom data generator. They receive a `GeneratorContext` (including the `faker` instance and schema field metadata) and return any value.

```typescript
// chameleon/generators/productCode.ts
import type { GeneratorContext } from "@chameleon/core";

export default function ({ faker }: GeneratorContext): string {
  return `PROD-${faker.string.alphanumeric(8).toUpperCase()}`;
}
```

### `data/`

Static JSON files used by overlays to deep-merge or replace specific route responses. See [Overlays](Overlays).

### `chameleon.config.ts`

The main configuration file. Use `defineConfig` for type-safety:

```typescript
import { defineConfig } from "@chameleon/core";

export default defineConfig({
  schemas: { openapi: "./chameleon/schemas/api.openapi.yaml" },
  annotations: "./chameleon/annotations/**/*.annotations.yml",
});
```

See [Configuration](Configuration) for the full reference.

## The `.chameleon/` directory

Generated by `chameleon build`. Contains `manifest.json` — a self-contained artifact with all parsed schemas, resolved annotations, and bundled generators. This file is loaded by the Vercel function at cold-start. Do not commit it; it is listed in `.gitignore`.
