# Proxy Mode

Proxy mode lets you switch between Chameleon's mock responses and a real upstream API on a per-request basis — without restarting the server or changing any config.

This is useful for:

- **Incremental adoption** — use real responses for endpoints you trust, mock responses for ones not yet implemented.
- **Debugging** — compare mock vs. real response bodies side by side.
- **Testing** — run integration tests against real data while keeping unit tests on mocks.

---

## Configuration

Add the `proxy` section to your `chameleon.config.ts`:

```typescript
import { defineConfig } from "@chameleon/core";

export default defineConfig({
  schemas: {
    openapi: "./chameleon/schemas/api.openapi.yaml",
  },
  proxy: {
    upstream:   "https://api.example.com",
    modeHeader: "X-Chameleon-Mode",   // default
    proxyValue: "proxy",              // default
  },
});
```

| Option | Type | Default | Description |
|---|---|---|---|
| `upstream` | `string` | — | Base URL of the real API |
| `modeHeader` | `string` | `"X-Chameleon-Mode"` | HTTP header Chameleon reads to decide the mode |
| `proxyValue` | `string` | `"proxy"` | Header value that activates passthrough |

---

## Usage

Without the header, all requests are mocked:

```bash
curl http://localhost:3000/users
# → fake response generated by Chameleon
```

With the header set to the proxy value, the request is forwarded to the upstream:

```bash
curl -H "X-Chameleon-Mode: proxy" http://localhost:3000/users
# → real response from https://api.example.com/users
```

The header is stripped before the request is forwarded, so the upstream never sees it.

---

## Custom header and value

You can use any header name and trigger value:

```typescript
proxy: {
  upstream:   "https://api.example.com",
  modeHeader: "X-Mode",
  proxyValue: "real",
}
```

```bash
curl -H "X-Mode: real" http://localhost:3000/users
```

---

## How it works

1. Every incoming request is inspected for the `modeHeader`.
2. If the header value matches `proxyValue`, Chameleon forwards the request to `upstream` + the original path + query string.
3. The upstream response is returned as-is, including status code, headers, and body.
4. If the header is absent or has a different value, Chameleon generates a mock response as usual.

Proxy mode is implemented as a Hono middleware and runs before any route handler, so it works for all endpoints including GraphQL.

---

## Environment variable override

The upstream URL can be overridden at runtime without touching the config file:

```bash
CHAMELEON_UPSTREAM_URL=https://staging.api.example.com pnpm dev
```

The header name can also be overridden:

```bash
CHAMELEON_MODE_HEADER=X-Use-Real-Api pnpm dev
```
